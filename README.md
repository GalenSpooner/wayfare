# Wayfare

A click-through prototype for a low-commission booking marketplace for independent shuttle and van operators in Northern California.

**The idea:** small shuttle operators (wine country vans, Tahoe airport shuttles, etc.) run seats empty on scheduled routes all the time. Wayfare negotiates a wholesale rate on those specific empty seats, lists the operator alongside everyone else on the corridor, and passes the savings to riders — full price stays full price on every seat that would've sold anyway.

## What's in here

A single self-contained `index.html` — no build step, no dependencies, no backend. Open it directly in a browser or serve it as a static site.

It includes:
- A landing page with a live search widget
- Real result sets for five researched corridors (Napa & Sonoma, Lake Tahoe, Sacramento Valley, Central Coast, Eastern Sierra), each pulling from actual named operators in that market
- A full booking/checkout flow (demo only — no real payment is processed)
- An operator-facing page with a working sign-up form

## Running it

Just open `index.html` in a browser, or enable **GitHub Pages** for this repo (Settings → Pages → deploy from `main` branch, root) to get a live shareable link.

## Status

Prototype / pitch collateral — not a production app. No backend, no persistence, no real payments.
